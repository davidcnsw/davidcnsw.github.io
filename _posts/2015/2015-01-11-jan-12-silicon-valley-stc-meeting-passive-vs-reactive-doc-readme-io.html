If you're in the San Francisco Bay area (near Santa Clara), come join us Monday, January 12 at around 7pm for a presentation by Greg Koberger on passive versus reactive documentation, plus a demo of readme.com docs, which is a new cloud platform for REST API documentation that is extremely well-designed.

I saw Greg present at a Write the Docs meeting a few months ago and was impressed. He said he is designer who is obsessed about technical documentation, and has spent countless hours studying best practices for documentation. He used this information to design readme.com docs, which seems like a great platform for web-based REST APIs.

If you only come to one STC meeting this month, this is the one to attend!
